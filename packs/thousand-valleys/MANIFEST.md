# Cartridge: thousand-valleys

A self-contained pscale-block bundle — an upland drove-road settlement (missing
stock / ridge-lights mystery). Seed it into an empty beach (a local folder, or a
sub-beach on a shared host) to get a complete, playable world; dump it to
snapshot; reset it to roll back. Same proof as thornwood: a game is just a bundle
of blocks — the engine never changes, the cartridge is the data.

**GRIT-native.** Unlike thornwood, this cartridge carries **no `function:<world>`
directive**: the room pool's underscore points at the canonical `pscale:grit`
sentinel (the daemonless play-loop, bundled in bsp-mcp), and the world supplies
only its *resolution rules* (`rules:nomad`) and its *place* (`rules:thousand-valleys`
+ `spatial:thousand-valleys`). Two-tier = GRIT (loop) + NOMAD (rules). A per-world
directive copy is redundant with the sentinel, so it is omitted by design.

## The three legs (on the beach)

- **Source** — this directory, in git (`pscale-beach/packs/thousand-valleys/`).
- **Run-time** — the cartridge seeded into a beach. Locally a folder
  (`scripts/local-beach.mjs`); remotely the sub-beach
  `thousand-valleys.beach.happyseaurchin.com`. Play mutates the run-time;
  `pack-reset` restores it from this source. One beach = one reality.
- **Origin** — generated by `bsp-mcp-server/scripts/seed-world.ts`, then cleaned
  (rules localized, pool → `pscale:grit`, scene + character state reset) and
  dumped here 2026-06-26.

## Substrate (the engine this cartridge runs against)

- **bsp-mcp-server** `16db82e` (origin/main) — the walker + sentinel registry,
  incl. `pscale://grit`. (The seed-world generic-rules fix is on branch
  `claude/seeder-generic-rules-grit`, pending merge — not required to *run* this
  cartridge, only to *re-generate* a clean one.)
- **pscale-beach** `2ba2b72` (main) — the `/.well-known/pscale-beach` handler.
- Portable across any beach: internal star-tags use the `{{BEACH}}` placeholder,
  substituted at seed. No absolute host refs in the cartridge.

## Contents

**`definition/`** — designer + author content; each `_`-locked under `VALLEYS_GM`:

| Block | Role |
|---|---|
| `rules:nomad` | resolution system — CF + SF + dice − difficulty, bands |
| `rules:thousand-valleys` | place physics — the Drover's Common, the settlement ground, the high country (drove-craft) |
| `spatial:thousand-valleys` | the world — the Drover's Common, stock-pens, Old Pell's counting-house, the Far-ridge Path, Sanna's cottage, the gather-stone, the grain-store |
| `frame-spec:thousand-valleys` | CADO manifest + frame metadata (unused under GRIT; kept clean) |

(No `function:thousand-valleys` — the room runs on `pscale:grit`.)

**`initial/`** — the opening save; characters are **unlocked** (open passports):

| Block(s) | Role |
|---|---|
| `passport:<h>` | capability (CF), wants, location, identity |
| `shell:<h>` | the character face (persona) + the manifest of the handle's blocks |
| `witnessed:<h>` | the private floor-1 spine — opens at the arrival beat only |
| `knows:<h>` | naming index — people (`1`) and places (`2`) known on arrival |
| `purpose:<h>` | the active drive (the agency lever) |
| `stats:<h>` | the NOMAD sheet — capability + condition (intact) |
| `pool:thousand-valleys-commons` | the room's public stream — underscore = `pscale:grit`; opens empty |
| `liquid:pool:thousand-valleys-commons` | the staging window (opens empty) |

Characters (6), all co-located at the Drover's Common (`spatial:1`) — they meet on arrival:
- **townsfolk** (here on business): **orvel** (livestock factor), **tessavar** (traveller / map-reader), **sable** (chronicler).
- **adventurers** (approaching the settlement, looking for adventure beyond the town): **cadell** (a Gal spear-for-hire whose war-band broke apart), **aderyn** (a Gal hunter / path-finder drawn by the ridge-lights), **yarem** (a nomad wanderer from the far grasslands beyond the southern mountains, broken from his tribe — *a barab*, the Gal word for the un-settled nomad-folk; the one outsider among the Gal).

## Lock policy (secret *values* are out-of-band — env vars, never committed)

| Blocks | Position | Secret env |
|---|---|---|
| all `definition/` | `_` | `VALLEYS_GM` |
| each character's blocks (`passport`/`shell`/`witnessed`/`knows`/`purpose`/`stats`) | `_` | `TV_<HANDLE>` — per-character (e.g. `TV_ORVEL`, `TV_YAREM`); each player gets only their own |
| `pool:` / `liquid:` | (open) | — |

Live value on `thousand-valleys.beach.happyseaurchin.com`: `VALLEYS_GM=valleys142`.

## Harness (run from `pscale-beach/`)

```sh
export VALLEYS_GM=valleys142
# a beach (local folder):
node scripts/local-beach.mjs --dir ./.beaches/thousand-valleys --port 8788 &
# seed / dump / reset:
node scripts/pack-seed.mjs  --beach http://localhost:8788 --pack packs/thousand-valleys
node scripts/pack-dump.mjs  --beach http://localhost:8788 --out ./snap --rewrite-beach http://localhost:8788
node scripts/pack-reset.mjs --beach http://localhost:8788 --pack packs/thousand-valleys

# reset the LIVE world (total rollback to this source):
VALLEYS_GM=valleys142 node scripts/pack-reset.mjs \
  --beach https://thousand-valleys.beach.happyseaurchin.com --pack packs/thousand-valleys
```
